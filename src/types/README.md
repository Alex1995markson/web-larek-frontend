# `src/types` — минимальный набор типов (data-only)

Этот пакет содержит **только типы данных**, без интерфейсов классов моделей/представлений, в соответствии с рекомендациями:

- Исключены интерфейсы классов (моделей, вью, презентера), чтобы снизить риск ошибок типизации на раннем этапе.
- Исправлена опечатка поля адреса: `address` (вместо `adress`).
- Эвенты адаптированы под **единый класс модального окна** (подход, используемый в проекте «Оно тебе надо»): введены события `modal:open`/`modal:close` с дискриминантным полем `kind`.
- DTO `CardDTO` временно выровнен с доменной моделью, чтобы избежать лишних трансформаций до стабилизации контракта.

## Содержимое

- `domain.ts` — доменные сущности и проекции: `ICard`, `IForm`, `TPayment`, `TContacts`, `TPurchase` и др.
- `api.ts` — DTO и **функциональные сигнатуры** API-клиента (`GetFn`, `PostFn`), без описания классов.
- `events.ts` — перечисление событий и карта нагрузок, совместимая с единым модальным окном.
- `index.ts` — barrel-экспорт.

## Импорт

```ts
import { ICard, TPayment, AppEventName, AppEventPayloads, ApiFns } from '@/types';
```

> На следующем этапе можно добавить реализации и, при необходимости, расширить типы, сохраняя обратную совместимость.
